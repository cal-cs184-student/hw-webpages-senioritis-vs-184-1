Task 1: Drawing Single-Color Triangles

I rasterize triangles by iterating over its surrounding box, checking each point for whether it meets the three-line test requirement (either all ≤ 0 or all ≥ 0 to account for the two possible winding orders). Finally, if the three-line test condition is met, the pixel in the sample buffer is filled with the color passed in as an argument. In my implementation, I calculate the max and min of the passed in vertex coordinates, which form a tight box around the triangle. I then only iterate within these bounds. Since I calculate the bounding box directly in my code, my algorithm by definition is no worse than one that checks each sample within the bounding box of the triangle.

Task 2: Antialiasing by Supersampling

In my supersampling algorithm, rather than iterating over entire pixels at once, I iterate over the subpixels and color in the corresponding subpixel in the sample buffer if the three line test condition is met. Afterwards, when resolving to framebuffer, I iterate over each pixel's subpixels and average the colors of each subpixel to obtain the overall pixel's color. Finally, the color is converted from the sample buffer's format to the format expected by the rgb frame buffer. Supersampling is useful because standard sampling could lead to aliasing in the output image (e.g. jaggies, broken tips), which can be unpleasant or confusing to the user. To implement supersampling, I had to adjust all references to the sample buffer to incorporate the sample rate, as each cell has to be referred to with an integer value. In addition to the modifications to the loops mentioned previously, I added the averaging portion of supersampling into resolve_to_framebuffer, so averaging is automatically done when the method is called. Supersampling allows for "blurring" of images, making features such as jaggies and broken tips less prominent. For example, a red triangle with jaggies will have its edges blurred with lighter reds by supersampling, making the jaggies less prominent.

Task 4: Barycentric coordinates

Barycentric coordinates allow us to interpolate positions or values within a triangle using a linear combination of three known positions or values. The vertices of the triangles are used to calculate weights for the three known values. We use barycentric coordinates to combine the colors of these vertices according to some calculated weights to determine what color each point in the triangle should be.

Task 5: "Pixel sampling" for texture mapping

Pixel sampling is the selection of pixels from an image to represent the image. To perform texture mapping, I first use the pixel's coordinates to generate alpha, beta, and gamma values. If the sampled point from the pixel meets the three line test requirement, these calculated values are used with the u, v values corresponding to each of the triangle's vertices to generate a sampled color for the sample buffer. Two types of sampling are supported: nearest sampling and bilinear sampling. In nearest sampling, the sampled color is that of the texture pixel closest to the provided u,v values. Bilinear sampling looks for the four texture pixels closest to the provided u,v values and uses them to perform multiple linear interpolations, after which the resulting texture pixel color is returned.

There will be a large difference between nearest and bilinear sampling in images with very thin, high color contrast details similar to the Campanile example above. This happens because nearest sampling only takes the texture color of its one closest neighbor whereas bilinear is able to use the four closest pixels on top of linear interpolation, which enhances its ability to produce intermediate colors. It's similar to supersampling, where the more samples you take per pixel, the better the blurring.

Task 6: "Level Sampling" with mipmaps for texture mapping

Level sampling is similar to the sampling mentioned in Task 5, except instead of always sampling textures from the original, full resolution texture, we sample textures from various mipmap levels. This was implemented for texture mapping determining which level the passed in partial derivatives corresponded to according to a mathematical formula. Then, when performing nearest or bilinear sampling, this calculated level would be used to determine which mipmap level to obtain a texture pixel color from.
